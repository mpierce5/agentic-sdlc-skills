# Publication Checklist

Run this before pushing or tagging a public/shared release.

- Search for personal identifiers, local usernames, and absolute machine paths.
- Search for employer, client, customer, project, product, and domain names.
- Search for credential-shaped content, including access credentials, service credentials, provider identifiers, private-key material, and real contact addresses.
- Confirm all URLs are generic examples or documented public project links intentionally meant to be published.
- Confirm model names are expressed as generic tiers unless they live in an adapter.
- Confirm platform-specific tool names live in adapters or are described as optional capabilities.
- Confirm examples use placeholder names, placeholder paths, and placeholder domains.
- Confirm repo-specific commands, ports, deployment lanes, and test accounts are not in shared skills.
- Confirm generated files, local caches, logs, and private planning artifacts are not staged.
- Review diffs manually before publishing.
